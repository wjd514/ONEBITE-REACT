# 1.4 변수와 상수
 1. 변수 : 프로그램이 실행되는 도중 계속 값을 바꿔가며 저장할 수 있는 저장소
  - 초기값을 설정해 주지 않아도 된다.
  - 특수한 경우 아니면 중복선언 불가.
  - ex. let
 2. 상수 : 값을 저장할 수 있는 저장소
  - 변수와 달리 한번 저장한 값은 변경 할 수 없다
  - 초기화 반드시 필요
  - ex. const
 3. 변수 명명규칙(네이밍 규칙)
  - $, _ 제외한 기호는 사용할 수 없다.
  - 숫자로 시작할 수 없다.
  - 예약어를 사용할 수 없다.

4. 변수 명명 가이드
 - 누가봐도 알 수 있도록 의미있는 변수명으로 명명하는게 좋다.

# 1.5 자료형
 - JavaScript의 자료형(Data Type)
 ```mermaid
 graph TD;
    A[자료형] --> B[원시 타입]
    A --> C[객체 타입]
    B --> B1[Number]
    B --> B2[String]
    B --> B3[Boolean]
    B --> B4[Null]
    B --> B5[Undefined]
    C --> D[Object]
    D --> D1[Array]
    D --> D2[Function]
    D --> D3[RegExp]
```
 1. 원시 타입
  - 기본형 타입이라고 불림
  - 프로그래밍에 있어 가장 기본적인 값들의 타입을 의미함
 1.1. Number Type
  - 존재할 수 있는 모든 숫자 값들을 포함한다
  - 기본적인 사칙연산들을 모두 지원
  - %와 같은 모듈러 연산도 지원
  - 양의 무한대, 음의 무한대와 같은 값(Infinity, -Infinity)도 지원
  - NaN(Not a Number) 도 Number Type에 포함
 1.2. String Type
  - 문자열 값들을 모두 포함하는 타입
  - 덧셈 연산 지원
  - 템플릿 리터럴 문법 : \`(백틱)은 작은 따옴표와 다른 따옴표로 기본적으로는 따옴표로 만든 문자열과 동일하지만 `${myName}`와 같이 동적으로 넣을 수 있다.
 1-3. Boolean Type
  - 참이나 거짓만을 담는 타입
 1-4. Null Type
  - 아무 것도 들어있지 않다를 표현하기 위해 사용된다
 1-5. Undefined Type
  - 변수를 선언하고 그 변수에 어떠한 값도 집어넣지 않았을 때 자동으로 할당되는 값
  - null 은 개발자가 직접 값을 줘야하지만 Undefined 는 값을 아무것도 주지 않았을 떄의 값이다

# 1.6 형 변환(Type Casting)
 1. 형 변환이란 : 어떤 값의 타입을 다른 타입으로 변경하는 것을 말함
 2. 묵시적 형변환 
  - 개발자가 설정하지 않아도 알아서 자바스크립트 엔진이 형 변환 하는것을 말함
  - 숫자와 문자를 더하는 것 같이 불가능한 연산시 자바스크립트에서 오류를 일으키지 않기 위해 묵시적으로 형 변환하는 것
 3. 명시적 형변환 
  - 개발자가 직접 내장함수 등을 이용해 형 변환을 일으킴
  - 내장함수란 자바스크립트가 기본적으로 제공하는 함수(Number, parseInt, String 등) 
  - parseInt 는 숫자값이 아닌 문자열이 있어도 변환해준다 ex. 10개 -> 10

# 1.7 연산자 1
 - 연산자(Operator) : 프로그래밍에서의 다양한 연산을 위한 기호, 키워드를 말함
 1. 대입 연산자 : 변수의 값을 저장할 떄 사용하는 가장 기초적인 연산자 ex. =
 2. 산술 연산자 : 5개의 연산(+,-,*,/,%)을 하는 연산자
 3. 복합 대입 연산자 : 산술 + 대입 연산자 ex. +=
 4. 증감 연산자 
  - 값을 1씩 늘리거나 줄일때 사용하는 연산자 ex. ++, --
  - 전위 연산 : ++num1 과 같이 연산자가 앞에 오는 연산으로 즉시 연산한다
  - 후위 연산 : num1++ 과 같이 연사자가 뒤에 오는 연산으로 라인이 바뀌며 연산한다.
 5. 논리 연산자
  - true, false를 표현할 떄 사용하는 연산자
  - ||, &&, ! 의 3가지 타입이 있다.
 6. 비교 연산자
  - 두개의 값을 서로 비교하는 연산자
  - 자료형까지 비교가 필요하면 ===, !== 을 사용하며 권장된다.

# 1.8 연산자 2
 1. null 병합 연산자
  - 존재하는 값을 추려내는 기능
  - null, undefined가 아닌 값을 찾아내는 연산자
 2. typeof 연산자
  - 변수의 데이터 타입을 반환하는 연산자
 3. 삼항 연산자
  - 항을 3개 사용하는 연산자
  - 조건문을 이용해서 참, 거짓일 떄의 값을 다르게 반환

# 1.9 조건문
 - 조건문(Conditional Statement) : 특정 조건을 만족했을 때에만 실행되는 코드를 작성하기 위한 문법 (if,switch...)
 1. if 조건문
  - 조건을 주고 참일떄 실행될 코드와 거짓일 떄 수행할 코드를 넣어서 실행하는 문법
  - if 로 시작해서 else 로 끝나야하며 중간에 else if는 얼마든지 들어가도 된다.
 2. switch 조건문
  - if문과 기능 자체는 도일
  - 다수의 조건을 처리할 때 if문 보다 직관적이다
  - 멈추고 싶을 때는 break; 를 사용
  - 어디에도 포함되지 않는 조건은 default를 사용한다.

# 1.10 반복문
 - 반복문(Loop, Iteration) : 어떠한 동작을 반복해서 수행할 수 있도록 만들어 주는 문법
 - for(초기식; 조건식; 증감식){} 으로 사용한다
 - 초기식 : 반복문 내에서만 쓸 수 있는 변수로 counting시 사용
 - 조건식 : 언제까지 반복할지 조건을 주는 식으로 참일시 반복, 거짓일시 종료
 - 증감식 : 초기식의 값을 증가시킬때 사용
 - break : 반복 중 사용하여 중간에 중지 시킬 수 있다
 - continue : 반복 중 사용하여 다음값으로 건너 뛸 수 있다.

# 1.11 함수
 - 중복으로 작성된 유사한 기능을 하는 코드를 묶어서 이름을 가지게 하여 해당 기능이 필요할 때 간단하고 편하게 기능을 사용할 수 있게 하는 문법
 - 함수 선언 : 함수를 새롭게 만드는 행위
 - 함수 호출 : 함수 선언 뒤 실행하기 위한 행위로 호출시 소괄호를 함께 붙여줘야 한다.
 - return 을 통해 반환할 수 있으며 return 후에는 코드가 있어도 동작하지 않는다.
 - 함수안에 또다른 함수를 넣을 수 있다.
 - 호이스팅 : 끌어올리다라는 뜻으로 함수가 뒤에 있어도 최상단으로 끌어올려서 실행하기 때문에 함수가 아래에 있어도 된다.

# 1.12 함수 표현식과 화살표 함수
 - 함수를 변수에 담아서 변수의 이름으로 호출 가능하다.
 1. 함수 표현식 : 함수를 변수에 담아서 선언하는 것으로 함수의 이름으로는 호출이 불가능하며 변수명으로만 호출 가능하다.
  - 함수 표현식으로 만든 함수의 경우 호이스팅이 되지 않는다.
 2. 화실표 함수 : 함수 표현식에서 function과 함수명 대신 let a =() => {} 만으로 함수를 선언할 수 있다.
  - 값을 반환하기만 한다면 {} 도 생략가능하다 let a = () => 1;

# 1.13 콜백 함수
 - 콜백 함수(Callback Function) : 자신이 아닌 다른 함수에, 인수로써 전달된 함수를 의미함
 - 반복해서 만드는 비슷한 함수를 간결하게 할 수 있다.

# 1.14 스코프
 - 스코프(Scope) : 범위를 뜻하며 변수나 함수에 접근하거나 호출할 수 있는 범위를 말함
 1. 전역(전체 영역) 스코프 
  - 전역 스코프는 코드의 모든 범위에서 사용 가능
 2. 지역(특정 영역) 스코프
  - 지역 스코프는 특정 코드 블록 내에서만 사용 가능
- 함수 선언식은 함수 안에서만 지역스코프를 갖으며 if나 for문 안에서는 갖지 않는다.

# 1.15 객체 1
 - 객체(Object) : 원시 타입이 아닌 객체 타입의 자료형
 - 여러가지 값을 동시게 저장할 수 있는 자료형을 의미
 ```mermaid
 graph TD;
    A[자료형] --> B[원시 타입]
    A --> C[객체 타입]
    B --> B1[Number]
    B --> B2[String]
    B --> B3[...]
    C --> D[Object]
    D --> D1[Array]
    D --> D2[Function]
    D --> D3[RegExp]
  ```
 - 객체를 이용하면 현실세계에 존재하는 어떤 사물이나 개념을 표현하기 용이함 
 1. 객체 생성
  - 객체 생성자 : let obj = new Object();
  - 객체 리터럴 : let obj2 = {}; (대부분 이 방식으로 사용)
 2. 객체 프로퍼티(객체 속성)
  - key : value 형식으로 되어있다.
  - property 의 value에는 boolean 타입이나 함수 타입이나 객체등도 들어갈 수 있다.
  - 숫자값도 key로 사용할 수 있다.
  - key에는 따옴표를 할 필요는 없지만 띄어쓰기가 포함된 key에는 예외적으로 따옴표를 붙인다.("like cat" : true, ...)
 3. 객체 프로퍼티를 다루는 방법
  1. 점 표기법 : let name = persion.name;
   - 존재 하지 않는 property 값을 호출시 undefined 발생
   - 새로운 프로퍼티 추가 방법 : person.job = "도적";
   - 프로퍼티 수정하는 방법 : person.age = 20;
   - 프로퍼티 삭제하는 방법 : delete person.job;
   - 프로퍼티 존재 여부 확인하는 방법 (in 연산자) : let result1 = "name" in person;
  2. 괄호 표기법 : let name = person["name"];
   - 쌍따옴표를 안붙일 시 변수로 인식된다. 
   - 새로운 프로퍼티 추가 방법 : person["favoriteFood"] = "라면";
   - 프로퍼티 수정하는 방법 : person["hobby"] = "게임";
   - 프로퍼티 삭제하는 방법 : delete person["extra"];
   - 프로퍼티 존재 여부 확인하는 방법 (in 연산자) : let result2 = "cat" in person;

# 1.16 객체 2
 1. 상수 객체
  - 상수에 저장된 객체는 추가,수정,삭제가 가능하다
  - 상수에 아예 새로운 값을 할당하는 것은 문제가 되지만 상수에 저장된 객체값의 프로퍼티를 수정하는 것은 가능하다.
 2. 메서드
  - 값이 함수인 프로퍼티를 말함
  - 객체 안에 메서드를 선언할 수 있으며 프로퍼티와 동일하게 호출가능하다.
  - person.sayHi(), persion["sayHI"]();
  - 메서드는 객체의 동작을 정의할 떄 주로 사용한다.

# 1.17 배열
 - 여러개의 값을 순차적으로 담을 수 있는 자료 형
 ```mermaid
 graph TD;
    A[Array] --> B["값(데이터)"]
    B --> C["떡볶이"]:::box
    B --> D["초콜릿"]:::box
    B --> E["삼겹살"]:::box
    B --> F["..."]
  ```
 1. 배열 생성
  - 배열 생성자 : let arrA = new Array();
  - 배열 리터럴 (대부분 사용) : let arrB = [];
  - 어떤 형식의 값도 들어 갈 수 있으며 길이 제한 또한 없다
  - 배열은 순차적으로 저장하기때문에 번호로 접근할 수 있으며 0부터 시작한다.
   - let item1 = arrC[0];
  - 특정 원소의 값을 수정도 가능하다.
   - arrC[0] = "Hello";
